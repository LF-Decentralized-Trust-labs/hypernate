# Contributing to Hypernate

Thank you for your interest in contributing to Hypernate!

The full, up-to-date contribution guide lives on our documentation site. Please start there:

**➡️ [Hypernate documentation](https://lf-decentralized-trust-labs.github.io/hypernate/)**

Open the **Contributing** section in the navigation, which covers:

- Reporting issues (bugs and feature requests)
- Working with the code (building, testing, and code style)
- Submitting pull requests (commits, sign-off, and review)
- Contributing to the documentation

## Quick reference

- **Start from an approved issue.** Every change — including trivial fixes — must be linked to a
  `design/approved` issue before you open a PR (enforced by an automated check).
- **Sign off your commits.** Every commit must be signed off (`git commit -s`) to certify the
  [Developer Certificate of Origin](https://developercertificate.org/).
- **Use Conventional Commits.** Follow the
  [Conventional Commits](https://www.conventionalcommits.org/en/v1.0.0/#summary) format.
- **Build before you push.** Run `./gradlew build` so formatting, compilation, and tests pass.
- **Be respectful.** Follow the
  [LF Decentralized Trust Code of Conduct](https://www.lfdecentralizedtrust.org/code-of-conduct).
